Responsive

By default, all th and td elements are hidden on small screen sizes.

Add the class responsive to table.

Add the class essential to the th of each column you want displayed on screen sizes smaller than 767px wide (for instance, iPhone portrait and landscape).

Add the class optional to the th of each column you want displayed on screen sizes between 768px and 979px wide (for instance, iPad portrait and landscape).

Note This component does not function on tables combining cells and rows with colspan and rowspan.

<table class="table responsive">
  <thead>
    <tr>
      <th>#</th>
      <th class="essential">First Name</th>
      <th class="essential">Last Name</th>
      <th class="optional">Username</th>

Description

A container, or module, is used to wrap a component and provide structure and basic styling for appearance within a page.

Containers have four primary elements: wrapper, header, body and footer. Each of the latter three elements function as wrappers for component-specific content, such as component title and actions, the component body, and supplemental links placed below the component as a footer.

Containers do not have set dimensions or margins. By default, they will stretch horizontally to fill their parent element and expand vertically to expose all content within.

Usage
<section class="module">
  <header class="module-header">
    …
  </header>
  <div class="module-body">
    …
  </div>
  <footer class="module-footer">
    …
  </footer>
</section>

Additional classes may be applied to the container wrapper .module to trigger additional styles or behavior. These classes may be combined as needed.

Class Usage
actionable For use when an actions dropdown is included in the container header.
chromeless Prevents the header and footer from displaying and removes all default background and padding rules.
draggable

Fixes the height of the container, centers the body content vertically within the container and applies hover/active styles to the header suitable for drag and drop interactions.

For use on Dashboard Views with the following script:

$( ".row" ).sortable(
  {
    handle:          '.module-header',
    helper:          'clone',
    items:           '.draggable',
    forceHelperSize: true,
    revert:          200,
    tolerance:       'pointer'
  }
);
Script Requirements:
  • jquery-ui
scrollable Fixes the height of the container and triggers content scrolling within the container body.

Description
Icons use Font Awesome to generate vector-based glyphs that inherit the size and color of the element they are placed within.
Usage

Icons use an <i> tag for all variants, but they have no case class—only a shared prefix. To use, place the following code just about anywhere:

<i class="icon-search"></i>

When using beside strings of text, as in buttons or nav links, be sure to leave a space after the <i> tag for proper spacing.
